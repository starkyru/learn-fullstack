JavaScript & runtime

ClosureA function bundled with the scope it was created in, so it keeps access to those variables after the outer call returns. MDN · module 02
Event loopThe runtime's scheduler: run a task, drain all microtasks, render, repeat. Explains why Promise callbacks beat setTimeout. MDN · module 02
Microtask / macrotaskMicrotasks (promises, queueMicrotask) run to exhaustion before the next macrotask (timers, I/O). MDN · module 02
Hoisting / TDZvar/functions are hoisted; let/const exist but are unusable in the temporal dead zone until declared. MDN · module 01/02
Async iterator / generatorfor await…of over a stream of promises; yield* delegates. MDN · module 02
Debounce / throttleDebounce fires after quiet; throttle fires at most once per interval. Both tame bursty events. MDN · module 07

TypeScript

Structural typingTypes match by shape, not name — if it has the fields, it fits. TS docs · module 01
GenericsType parameters that let a function/type work over many types while staying checked. TS docs · module 01
Discriminated unionA union with a common literal tag so narrowing on the tag picks the variant. TS docs · module 01
Type guard / narrowingA runtime check (typeof, in, predicate) that refines a type in a branch. TS docs · module 01

React

ReconciliationReact diffs the new element tree against the old and applies minimal DOM changes; key identity drives it. React · module 05/08
HookA function (use*) that hooks into React state/lifecycle; rules: top-level only, same order every render. React · module 06/07
MemoizationuseMemo/useCallback/React.memo skip work when inputs are referentially stable. React · module 07/08 · REACT_PERFORMANCE
PortalRender children into a DOM node outside the parent (modals, toasts). React · module 07/11
SuspenseA boundary that shows a fallback while a child "suspends" on data/code. React · module 10
TransitionuseTransition/useDeferredValue mark updates low-priority so input stays responsive. React · module 10
External storeuseSyncExternalStore subscribes React to an outside source (socket, store) with a stable snapshot. React · module 13/22

Rendering & Next.js

CSR / SSR / SSG / ISRRender in the browser / on each request / at build / rebuilt-on-a-timer. Next.js · module 23/24
HydrationAttaching React event handlers/state to server-rendered HTML to make it interactive. React · module 23
RSC (Server Component)A component that runs only on the server, ships no JS; "use client" marks the interactive boundary. React · module 23/24
Server ActionA server function callable from the client as a form action / mutation. Next.js · module 25

State & data

Flux / unidirectionalState flows one way: action → reducer → store → view. Redux is the canonical shape. Redux · module 12/13
Server state / cacheAsync remote data managed separately from UI state — fetch, cache, invalidate. TanStack Query · module 14
Optimistic updateApply the change in the UI immediately, roll back if the server rejects. TanStack Query · module 10/14
Cache invalidationMarking cached data stale after a mutation so it refetches (invalidateQueries). TanStack Query · module 14

Backend & API

RESTResources + HTTP verbs + status codes; described by OpenAPI, codegen'd into clients. MDN HTTP · module 19
GraphQLOne typed endpoint; the client asks for exactly the fields it needs. graphql.org · module 20
ResolverThe function that produces a field's value in a GraphQL schema. Apollo · module 20
DataLoader / N+1Batches + caches per-request lookups to kill the N+1 query storm. DataLoader · module 20b
DI (Nest)The framework constructs and injects dependencies; you declare providers, not new. NestJS · module 18
Guard / Interceptor / PipeNest request stages: authorize / wrap & transform / validate & coerce. NestJS · module 18
IdempotencySame request applied twice = same effect once; keyed so retries don't double-charge. MDN · module 19/22b
WebhookServer→server HTTP callback on an event; signed, retried, deduped, replayable. · module 22b

Database

ORMMaps rows ↔ typed objects; Prisma generates a client + migrations from a schema. Prisma · module 16
MigrationA versioned, ordered schema change applied safely forward (and ideally reversible). Prisma · module 16
Transaction / ACIDA group of statements that commit all-or-nothing with isolation guarantees. Postgres · module 15
Index / EXPLAINA lookup structure that speeds reads; EXPLAIN ANALYZE shows the query plan. Postgres · module 15/28
Connection poolReuse a set of open DB connections instead of one per request. node-postgres · module 15/16

Auth & security

Session vs JWTOpaque server-stored id vs signed self-contained claim. Full comparison → AUTH_COMPARISON.html · module 21/21b
OAuth2 / OIDCDelegated authorization + an identity layer; Authorization Code + PKCE. oauth.net · module 21/25
CSRFForged cross-site request riding a cookie; stop with SameSite + double-submit token. MDN · module 21/21b
XSSInjected script running in your page; stop by escaping output + CSP + not storing tokens in JS. MDN · module 21
RBAC / ABACAuthorize by role vs by attributes (owner, org, context). OWASP · module 21

BackoffExponential wait between reconnect/retry attempts to avoid hammering. MDN · module 22/22b
